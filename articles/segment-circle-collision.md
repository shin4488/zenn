ゲームを作っていると、線分と円の当たり判定が欲しくなる場面がちょくちょくあります。レーザーが敵に当たったかどうかはまさにこれですし、速い弾のすり抜け対策も、弾が1フレームで通った道を線分とみなして敵の円と判定する話になります。

円同士の当たり判定は、中心間の距離を半径の和と比べるだけで書けます。実は線分と円の当たり判定もやることは同じです。「円の中心から線分までの最短距離を測って、半径と比べる」。これがゴールです。

![円同士と線分と円の当たり判定の比較](/images/segment-circle-collision/fig-intro.png)
*どちらも「距離」と「半径」を比べるだけ。違いは距離の測り方*

ただ、この最短距離をどうやって測るのか。そこで出てくるのが三角関数とベクトルです。前半で測り方を考えて、後半で TypeScript のコードに落とします。使う数学は三角関数と三平方の定理だけです。

## 三角関数のおさらい

円の中心から線分までの最短距離は、sinθ を使って測ります。その準備として、sin と cos の意味を軽くおさらいしましょう。

長さ L の棒を、角度 θ だけ傾けて地面に立てたとします(下の図の左側)。このとき、根元から先端までの横方向の距離は $L \times \cos\theta$ になります。数式では掛け算の記号を省略して $L\cos\theta$ と書くので、この記事でも以降はそう書きます。同じように、高さは $L\sin\theta$ です。棒を寝かせるほど横に伸びて、立てるほど高くなります。つまり、

- 斜めの長さ × cosθ =「横」の長さ
- 斜めの長さ × sinθ =「縦」の長さ

この記事で使うのはほぼこれだけです。

![sinとcosの意味](/images/segment-circle-collision/fig0-sincos.png)
*左:斜めの長さ L は、横 $L\cos\theta$ と縦 $L\sin\theta$ に分けられる。右:90° を超えると横はマイナスになる*

もうひとつ、右側の図も見てください。θ が 90° を超えると先端が根元より後ろに回り込むので、横方向の距離 $L\cos\theta$ はマイナスになります。つまり **cosθ の符号を見れば、先端が「前」にあるか「後ろ」にあるかがわかります**。こちらは後半、円が線分の端からはみ出した位置にいるかを見分けるところで使います。

## 内積・外積と三角関数

高さ $L\sin\theta$ で距離が測れるといっても、ゲームが持っている座標データは点の位置だけで、角度 θ はどこにも記録されていません。`Math.atan2` で θ を求める手もありますが、実はその必要すらありません。座標の掛け算と足し算だけで、cosθ・sinθ の掛かった値を直接取り出す方法があるからです。それがベクトルの内積と外積です。

計算はベクトル(点 A から点 B へ向かう矢印)で進めます。まず下の図のように、2本のベクトル A と B を根元を揃えて並べて、なす角(矢印の間の角度)を θ とします。B を「A に立てかけた、長さ $|B|$ の棒」と見ると、さっきの話の通り、B の横の長さは $|B|\cos\theta$、高さは $|B|\sin\theta$ です。

![ベクトルBの横の長さと高さ](/images/segment-circle-collision/fig1-dot-cross.png)
*B を A に立てかけた棒と見ると、横の長さが $|B|\cos\theta$、高さが $|B|\sin\theta$*

内積と外積は、この2つの長さとつながっています。A の長さを $|A|$ と書くと、

$$\text{内積:}\ A \cdot B = |A||B|\cos\theta \qquad \text{外積:}\ A \times B = |A||B|\sin\theta$$

が成り立ちます。内積からは B の「横の長さ」が、外積からは B の「高さ」がわかるということです($|A|$ 倍が付いていますが、$|A|$ で割れば長さそのものが出ます)。左辺の内積・外積を座標から計算する式は、実装の節で出てきます。掛け算と足し算だけの単純な式なので、座標さえあれば角度を経由せずにこの2つの長さが手に入ります。

あとは A と B に何を当てはめるかです。イメージだけ先に言うと、A を線分に、B を「線分の始点から円の中心への矢印」にすると、B の「高さ」がそのまま円までの距離になります。つまり外積で円までの距離が測れて、内積で円がどのあたりにいるかを見分けられます。詳しくは次の節から見ていきましょう。

## 円の中心から「直線」までの距離は外積でわかる

ここからゴールである「最短距離」を測りにいきます。線分の始点を S、終点を E、円の中心を C、半径を r とします。

いきなり線分を相手にすると大変なので、まずは S と E を通る「直線」(両側に無限に伸びた線)で考えます。円は中心から距離 r 以内の範囲のことなので、C から直線までの最短距離 d を測れば、d ≤ r かどうかで直線と円が重なっているかが決まります。最短距離は、C から直線へ垂直に下ろした線(垂線)の長さです。

![円の中心から直線までの距離d](/images/segment-circle-collision/fig2-distance.png)
*$d = |SC|\sin\theta$:円の中心から「直線」までの距離*

この d を、前の節の A と B に当てはめて測ります。A にあたるのが線分方向のベクトル SE、B にあたるのが始点 S から円の中心 C へ引いたベクトル SC です。図のように SC を「SE に立てかけた棒」と見ると、棒の高さがちょうど d になっています。つまり $d = |SC|\sin\theta$ です。

そして高さは、外積から取り出せるのでした。

$$|SE \times SC| = |SE||SC|\sin\theta$$

右辺をよく見ると、欲しい $|SC|\sin\theta$ に $|SE|$ が余分に掛かっているだけです。両辺を $|SE|$ で割れば、

$$d = |SC|\sin\theta = \frac{|SE \times SC|}{|SE|}$$

となって、座標から計算できる外積と長さだけで d が求まります。  
※ なお SE と SC そのものは座標の引き算で作れます。E の座標から S の座標を引けば SE、C の座標から S の座標を引けば SC です。詳しくは実装の節で見ます。

d > r なら、無限に伸びる直線ですら円に届いていないのだから、その一部でしかない線分も当たっていません。ここで大半のケースをふるい落とせます。

## 線分には端がある

では d ≤ r なら当たりかというと、そうとは限りません。下の図がその例です。直線は円と重なっていますが(d ≤ r)、線分は円に届いていません。

![dはr以下だが線分は円に届いていない例](/images/segment-circle-collision/fig3-short-segment.png)
*延長線上では重なっていても、線分そのものは届いていない*

直線は無限に伸びていますが、線分には両端があります。図のように、延長線上では円と重なっていても、線分自体は届いていないケースがあります。直線と円の最短距離の判定(d ≤ r)だけで実装を終えると、画面の端で撃った弾がはるか遠くの敵に当たる、というバグが起きます。

そこで、円の中心が S と E の間にいるのか、端からはみ出した外側にいるのかを区別する必要があります。

## 円の中心がどこにいるかは内積でわかる

ここで、最初におさらいした「90° を超えると cosθ はマイナス」を使います。

内積 $SE \cdot SC = |SE||SC|\cos\theta$ のうち、長さの部分は必ず 0 以上です。だから内積の符号は cosθ の符号と同じです。SE・SC がマイナスなら、C は S から見て線分と逆側、つまり始点の外側にいます。終点側も同じで、ES・EC がマイナスなら C は終点の外側です。

![内積の符号による3つの領域分け](/images/segment-circle-collision/fig4-regions.png)
*円の中心がどの領域にいるかは、内積の符号(cosθ の正負)でわかる*

- SE・SC < 0 → C は始点 S の外側
- ES・EC < 0 → C は終点 E の外側
- どちらも 0 以上 → C は S と E の間(図の中央の領域)

S と E の間にいるなら垂線はちゃんと線分の上に降りるので、d ≤ r がそのまま結論になります。外側にいる場合は、線分の中で C に一番近い点は端点そのものなので、端点と C の距離を r と比べれば判定できます。

考え方はこれで全部です。判定の流れをまとめると、こうなります。

```mermaid
flowchart TD
    Q1{"1. 外積で求めた垂線の距離 d は r 以下?"} -->|No| NG1["外れ"]
    Q1 -->|Yes| Q2{"2. C は S と E の間にいる?(内積の符号)"}
    Q2 -->|Yes| OK1["当たり"]
    Q2 -->|No| Q3{"3. 近いほうの端点との距離は r 以下?"}
    Q3 -->|Yes| OK2["当たり"]
    Q3 -->|No| NG2["外れ"]
```

## 実装

ここからは上の 1〜3 をコードにします。まずベクトルまわりの関数から。

```ts
// 2Dベクトル。ただの x, y のペア
type Vec = { x: number; y: number };

// ベクトルを作る
const vec = (x: number, y: number): Vec => ({ x, y });

// a から b へ向かうベクトル(b - a)
const sub = (b: Vec, a: Vec): Vec => vec(b.x - a.x, b.y - a.y);

// ベクトルの長さ(三平方の定理)
const len = (v: Vec): number => Math.hypot(v.x, v.y);

// 内積:|a||b|cosθ に一致する
const dot = (a: Vec, b: Vec): number => a.x * b.x + a.y * b.y;

// 外積:|a||b|sinθ に一致する(2Dでは数値が1つ返る)
const cross = (a: Vec, b: Vec): number => a.x * b.y - a.y * b.x;
```

内積も外積も、実装はこの掛け算と足し算だけです。なお長さの関数を `length` にしなかったのは、トップレベルに書くと DOM の型定義にある `window.length` と衝突してコンパイルエラーになるためです。

判定本体は、理論の 1〜3 をそのまま並べます。

```ts
// s, e: 線分の始点・終点  c: 円の中心  r: 半径
function isHitSegmentCircle(s: Vec, e: Vec, c: Vec, r: number): boolean {
  const se = sub(e, s); // 線分ベクトル
  const sc = sub(c, s); // 始点から円の中心へ

  // 1. 垂線の距離 d = |SE × SC| / |SE| で足切り
  const d = Math.abs(cross(se, sc)) / len(se);
  if (d > r) return false;

  // 2. 円の中心が S と E の間にいるなら、d ≤ r なので当たり
  const es = sub(s, e); // 終点から始点へ
  const ec = sub(c, e); // 終点から円の中心へ
  if (dot(se, sc) >= 0 && dot(es, ec) >= 0) return true;

  // 3. 外側にいるなら、近いほうの端点との距離で判定
  return len(sc) <= r || len(ec) <= r;
}
```

外積は円が線のどちら側にあるかで符号が変わるので、距離として使う 1 では `Math.abs` で絶対値を取っています。

## 動作確認

横一直線の線分 S(0, 0)-E(100, 0) に、位置と半径を変えた3つの円をぶつけてみます。

```ts
const s = vec(0, 0);
const e = vec(100, 0);

// ケース1:S と E の間、線分から距離30の位置に半径40の円
console.log(isHitSegmentCircle(s, e, vec(50, 30), 40));  // true

// ケース2:延長線上、端点Eから50離れた位置に半径40の円
console.log(isHitSegmentCircle(s, e, vec(150, 0), 40));  // false

// ケース3:ケース2と同じ位置で、半径だけ60に広げた円
console.log(isHitSegmentCircle(s, e, vec(150, 0), 60));  // true
```

それぞれの配置と、判定の根拠を図にするとこうなります。

![3つのテストケースの配置と判定根拠](/images/segment-circle-collision/fig5-testcases.png)
*ケース2と3は円の位置が同じで、半径だけが違う*

ケース1は円の中心が S と E の間にいるので、垂線の距離 d = 30 と半径 40 の比較になり、当たりです。

ポイントはケース2です。円の中心が延長線上にあるので d = 0 ですが、内積の分類が「終点 E の外側」と見抜くため、比較相手は端点との距離 |EC| = 50 に切り替わります。半径 40 では届かず、外れです。距離の足切りだけの実装だと、ここで誤って当たりにしてしまうところでした。ケース3は同じ位置でも半径が 60 に伸びた分だけ端点に届くので、当たりになります。

## 実戦での注意点

罠がひとつあります。線分の長さが 0 だと `len(se)` が 0 になり、0 除算が起きます。JavaScript の 0 除算は例外にならず Infinity や NaN が返るだけなので、静かに壊れます。移動前後の座標から線分を作る使い方だと「その場から動かなかったフレーム」で普通に発生するので、関数の先頭でガードしておきます。

```ts
if (se.x === 0 && se.y === 0) {
  // 線分が潰れているときは、点と円の判定に落とす
  return len(sc) <= r;
}
```

:::message
この判定が真価を発揮するのは高速な弾です。毎フレーム「点と円」で判定していると、速い弾はフレームの間に敵を通り過ぎて、すり抜けが起きます。前フレームの位置と現在位置を結んだ「線分」で判定すれば、通り道ごとチェックできるのですり抜けません。
:::

## まとめ

- 外積 $|SE \times SC| = |SE||SC|\sin\theta$ から、角度を計算せずに垂線の距離 d が出せる
- 内積の符号(= cosθ の符号)で、円の中心が S と E の間にいるか、はみ出しているかを分類できる
- 外側のときだけ、端点との距離で判定する

sin と cos の意味まで戻って考えると、当たり判定は公式の暗記ではなく、その場で組み立てられるものになります。カプセル(太さのある線分)と円の判定も同じ考え方の延長で作れるので、次の題材にちょうどいいと思います。
